# 🗄️ 轻量级关系型数据库引擎 - 项目进度汇报

## 一、题目介绍

### 1.1 项目背景

本项目旨在实现一个基于 **B+ 树索引** 的轻量级关系型数据库引擎，模拟真实数据库的核心存储机制。项目采用经典的三层架构设计，涵盖 SQL 解析、查询执行和存储引擎三个层次。

### 1.2 技术架构

```
┌─────────────────────────────────────────────────────────────┐
│                      SQL 解析层                              │
│  ┌──────────────┐    ┌──────────────┐                       │
│  │   Lexer      │───▶│   Parser     │                       │
│  │ 词法分析器   │    │ 语法分析器   │                       │
│  └──────────────┘    └──────┬───────┘                       │
└─────────────────────────────│───────────────────────────────┘
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                      查询执行层                              │
│  ┌──────────────┐    ┌──────────────┐                       │
│  │ Query Plan   │───▶│ Executor     │                       │
│  │ 查询计划生成 │    │ 查询执行器   │                       │
│  └──────────────┘    └──────┬───────┘                       │
└─────────────────────────────│───────────────────────────────┘
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                      存储引擎层                              │
│  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐   │
│  │ Buffer Pool  │───▶│  B+ Tree     │───▶│  Disk I/O    │   │
│  │ 缓冲池(LRU)  │    │ 索引结构     │    │ 持久化层     │   │
│  └──────────────┘    └──────────────┘    └──────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

### 1.3 核心技术点

| 技术点 | 说明 |
|--------|------|
| **B+ 树索引** | 高效的平衡树结构，支持快速插入、删除、范围查询 |
| **LRU 缓冲池** | 基于最近最少使用策略的页面缓存，减少磁盘 I/O |
| **页式存储** | 固定大小（4KB）的磁盘页面管理 |
| **位图管理** | 使用位图高效管理空闲页面 |

**目标**：实现基于LRU的页缓存

```c
// buffer_pool.c 核心功能
```

**设计要点**：

- 使用**哈希表**加速页查找
- 页帧结构：`data[PAGE_SIZE] + page_id + is_dirty + pin_count`

## 三、遇到的问题

### 3.1 B+树节点分裂问题

**问题描述**：插入数据时，节点分裂后右节点键数不足，导致后续删除操作出错。

**原因分析**：分裂条件设置不当，当节点有10个键（达到最大容量）时才分裂，导致右节点只有4个键（少于最小要求的5个）。

**解决方案**：调整分裂条件为 `key_count > BPT_MAX_KEYS`（即11个键时才分裂），确保分裂后左右节点各有至少5个键。

**文件**：`src/bplus_tree.c`

---

### 3.2 B+树根节点删除双重释放

**问题描述**：删除根节点时出现堆内存双重释放错误，程序崩溃。

**原因分析**：`delete_recursive` 函数在递归过程中错误地释放了根节点，而 `bpt_delete` 函数又尝试访问已释放的根节点。

**解决方案**：将根节点收缩逻辑从 `delete_recursive` 中移除，统一在 `bpt_delete` 函数中处理。

**文件**：`src/bplus_tree.c`

---

### 3.3 缓冲池哈希表冲突

**问题描述**：使用开放寻址法处理哈希冲突时，驱逐页面后无法正确重新哈希后续条目。

**原因分析**：开放寻址法在删除元素后需要重新插入冲突链上的后续元素，实现复杂且容易出错。

**解决方案**：改用链表法处理哈希冲突，每个哈希槽维护一个链表，插入和删除操作简单可靠。

**文件**：`src/buffer_pool.c`

---

### 3.4 LRU链表空指针处理

**问题描述**：当页面帧不在LRU链表中时，`lru_remove` 函数会错误地清空整个链表。

**原因分析**：`lru_remove` 没有正确处理孤立节点（prev和next都为NULL但仍在链表中）的情况。

**解决方案**：添加边界条件检查，当节点的prev和next都为NULL时，检查是否是链表中唯一的节点。

**文件**：`src/buffer_pool.c`

---

### 3.5 磁盘写入限制

**问题描述**：`dm_write_page` 只允许写入已分配的页面，无法写入超出文件大小的新页面。

**原因分析**：写入前检查 `page_id >= dm->num_pages`，导致新分配的页面无法写入。

**解决方案**：移除页面ID检查，允许写入超出当前文件大小的页面，并自动扩展文件。

**文件**：`src/disk_manager.c`
