## 1. 项目概述

### 1.1 项目背景

数据库引擎涵盖存储管理、索引结构和查询处理三大核心模块，是验证数据结构与操作系统知识的理想载体。本项目在纯 C 语言环境下实现一个支持 SQL 子集的单文件数据库。

### 1.2 项目目标

实现一个基于 B+ 树索引的轻量级关系型数据库，具备以下核心功能：

| 序号 | 功能 | 状态 |
|------|------|------|
| 1 | 固定大小 Page（4KB）文件管理 | ✅ |
| 2 | LRU 缓冲池替换策略 | ✅ |
| 3 | B+ 树索引（插入、删除、查找、范围扫描） | ✅ |
| 4 | SQL 解析器（CREATE TABLE/INSERT/SELECT/UPDATE/DELETE） | ✅ |
| 5 | Write-Ahead Log（WAL）事务日志 | ✅ |
| 6 | CLI 命令行界面 | ✅ |

## 4. 核心模块实现

### 4.1 B+ 树索引

#### 4.1.1 节点结构

```c
typedef struct BPlusTreeNode {
    int is_leaf;           // 0: 内部节点, 1: 叶子节点
    int key_count;         // 当前键数量
    int keys[BPT_MAX_KEYS_ARRAY];  // 键数组 (最大10个)
    struct BPlusTreeNode* children[BPT_MAX_KEYS_ARRAY + 1];  // 子节点指针
    struct BPlusTreeNode* next;   // 叶子节点链表指针
    struct BPlusTreeNode* prev;   // 叶子节点链表指针
    record_id_t records[BPT_MAX_KEYS_ARRAY];  // 记录ID（仅叶子节点）
} BPlusTreeNode;
```

#### 4.1.2 节点结构图

**内部节点**：
```
+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+
|is_leaf=0|key_count|  key[0]  |  key[1]  | ... |
+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+
| child[0] | child[1] | child[2] | ... | child[N] |
+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+
```

**叶子节点**：
```
+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+
|is_leaf=1|key_count|  key[0]  |  key[1]  | ... |
+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+
|record[0]|record[1]|record[2]| ... |record[N]|
+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+
|   prev   |   next   |
+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+
```

#### 4.1.3 核心操作

| 操作 | 函数 | 时间复杂度 |
|------|------|------------|
| 插入 | `bpt_insert()` | O(log n) |
| 删除 | `bpt_delete()` | O(log n) |
| 查找 | `bpt_search()` | O(log n) |
| 范围查询 | `bpt_range_query()` | O(log n + k) |

### 4.2 LRU 缓冲池

#### 4.2.1 数据结构

```c
typedef struct Frame {
    Page page;
    page_id_t page_id;
    int is_dirty;
    int pin_count;
    struct Frame* prev;
    struct Frame* next;
} Frame;

typedef struct {
    Frame* frames;
    void* hash_table;
    Frame* lru_head;
    Frame* lru_tail;
    DiskManager* dm;
    int num_frames;
    int max_frames;
} BufferPool;
```

#### 4.2.2 LRU 链表结构

```
lru_head                         lru_tail
    |                               |
    v                               v
+-------+ <-> +-------+ <-> ... <-> +-------+
| Frame |     | Frame |             | Frame |
| (新)  |     | Frame |             | (旧)  |
+-------+     +-------+             +-------+
```

#### 4.2.3 页面置换策略

1. 当缓冲池满时，淘汰 LRU 链表尾部的页面
2. 如果页面是脏页（is_dirty=1），先写回磁盘再淘汰
3. 被 Pin 的页面（pin_count > 0）不会被淘汰

### 4.4 WAL 事务日志

#### 4.4.1 日志记录格式

```c
typedef struct {
    uint64_t txn_id;      // 事务ID
    WalRecordType type;    // 记录类型
    uint64_t page_id;     // 关联页面ID
    uint32_t data_len;    // 数据长度
    char data[MAX_LOG_ENTRY_SIZE - 28];  // 数据内容
} WalRecord;
```

#### 4.4.2 日志类型

| 类型 | 说明 |
|------|------|
| WAL_INSERT | 插入操作 |
| WAL_UPDATE | 更新操作 |
| WAL_DELETE | 删除操作 |
| WAL_COMMIT | 事务提交 |
| WAL_ROLLBACK | 事务回滚 |

#### 4.4.3 崩溃恢复流程

```
1. 打开 WAL 文件
2. 读取 WAL Header，获取 last_txn_id
3. 遍历所有 WAL 记录
4. 对于每个记录：
   - 如果是 COMMIT 记录：标记事务已提交
   - 如果是 ROLLBACK 记录：标记事务已回滚
   - 如果是 INSERT/UPDATE/DELETE：
     - 如果事务已提交：应用变更到数据页面
     - 如果事务未提交：忽略该记录
5. 完成恢复
```

## 5. 算法流程

### 5.1 B+ 树插入算法

```
算法：bpt_insert(tree, key, rid)
输入：B+树指针、键值、记录ID
输出：无

1. 如果树为空：
   a. 创建根节点（叶子节点）
   b. 将键和记录插入根节点
   c. 返回

2. 从根节点开始，向下查找合适的叶子节点：
   a. 对于内部节点，找到第一个大于 key 的键
   b. 进入对应的子节点
   c. 重复直到到达叶子节点

3. 在叶子节点中插入键和记录：
   a. 找到插入位置（保持有序）
   b. 将键和记录插入

4. 如果节点溢出（key_count > BPT_MAX_KEYS）：
   a. 创建新的右兄弟节点
   b. 将节点后半部分移动到右兄弟
   c. 如果是根节点：
      i. 创建新根节点
      ii. 将中间键提升到新根
   d. 否则：
      i. 将中间键插入父节点
      ii. 将右兄弟作为新子节点添加到父节点
```

### 5.2 B+ 树删除算法

```
算法：bpt_delete(tree, key)
输入：B+树指针、键值
输出：无

1. 从根节点开始，向下查找包含 key 的叶子节点
2. 如果未找到，返回

3. 在叶子节点中删除键和记录

4. 如果节点键数不足（key_count < BPT_MIN_KEYS）：
   a. 尝试从左兄弟借键
   b. 如果左兄弟也不足，尝试从右兄弟借键
   c. 如果都无法借键，与兄弟节点合并
   d. 如果合并后父节点键数不足，递归处理父节点

5. 如果根节点键数为0且有子节点：
   a. 将根节点替换为唯一子节点
```

### 5.3 SQL 查询执行流程

```
SQL字符串 → Lexer词法分析 → Token流
                              ↓
                    Parser语法分析 → AST抽象语法树
                                       ↓
                              Executor查询执行
                                       ↓
                              B+树索引操作
                                       ↓
                              结果集返回
```

### 5.4 INSERT 执行流程

```
INSERT语句 → 解析表名和值 → 创建记录
                              ↓
                    将第一个整数值作为索引键插入B+树
                              ↓
                    将记录写入内存数组
                              ↓
                    返回成功
```

### 5.5 SELECT 执行流程

```
SELECT语句 → 解析表名和条件
               ↓
         有WHERE条件？
               ↓
         是 → 使用B+树索引查找
         否 → 全表扫描
               ↓
         返回匹配的记录集
```

## 7. 性能分析

### 7.1 B+ 树性能

**时间复杂度分析**：

| 操作 | 时间复杂度 | 说明 |
|------|------------|------|
| 插入 | O(log n) | 需要遍历树高，可能触发分裂 |
| 删除 | O(log n) | 需要遍历树高，可能触发合并 |
| 查找 | O(log n) | 二分查找定位 |
| 范围查询 | O(log n + k) | k 为结果数量 |

**树高计算**：
- 阶数：10（每个节点最多10个键）
- 10万条记录的树高：log₁₀(100,000) ≈ 5层

### 7.2 缓冲池性能

**命中率分析**：
- 缓冲池大小：10个帧
- 当访问模式具有局部性时，命中率高
- 全表扫描时，命中率低

### 7.3 性能优化方向

1. **调整 B+ 树阶数**：增大阶数减少树高
2. **增大缓冲池大小**：减少缺页次数
3. **实现谓词下推**：减少不必要的数据扫描
4. **缓冲池与 B+ 树集成**：实现持久化存储
